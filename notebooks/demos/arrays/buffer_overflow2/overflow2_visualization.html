Buffer Overflow Visualizer: password overflow corrupts valid

Teaching model for buffer_overflow2.cpp: memory is shown as username[10], password[10], then valid (1 byte). When password input is longer than 9 characters, operator >> keeps writing and can overwrite valid. Any non-zero byte in valid is treated as true, so access can be granted even with wrong credentials.

username buffer interpreted as string
password buffer interpreted as string
valid byte value (decimal) 
bool valid = false;
char username[10];
char password[10];

cin >> username;
cin >> password; // no bounds check

if (string(username) == "backdoor" and strcmp(password, "123456") == 0)
    valid = true;

if (valid) {
    cout << "Your SSN is 123-45-5678";
}

Real stack layout can vary by compiler/platform. This is a purpose-built classroom visualization to show how overflowing password can corrupt valid.
